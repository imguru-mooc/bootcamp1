# 임베디드 AI 기말 프로젝트 안내서
### 라즈베리 파이 4B + LED 기반 AI 추론 제어 시스템

> **플랫폼**: Raspberry Pi 4B · Ubuntu 22.04 LTS
> **하드웨어**: LED 1개 이상 (GPIO 연결) — *카메라·부저 등 추가 장치 없음*
> **AI 프레임워크**: LiteRT(`ai-edge-litert`)
> **평가 형태**: 개인별 / 8시간 / 문서·코드·시연 제출
> **연계 실습**: 라즈베리 파이 AI 실습 0–9 (전이학습 · 분류 · 탐지 · 세그멘테이션 · AI 기반 LED 제어)

## 1. 프로젝트 개요

이번 기말 프로젝트는 1주차에 학습한 **라즈베리 파이 + AI** 내용을 한 사람이 처음부터 끝까지 통합 구현하는 과제입니다. 핵심 파이프라인은 다음과 같습니다.

```
[이미지 파일]  →  [LiteRT AI 추론]  →  [추론 결과 해석]  →  [LED 제어]
   입력             분류/탐지/세그           로직 매핑          GPIO 출력
```

> 하드웨어가 **LED만** 주어지므로, 입력은 **카메라가 아닌 이미지 파일**을 사용합니다. AI가 이미지를 추론한 결과(클래스 · 확신도 · 개수 · 면적 비율 등)를 LED의 점등 · 점멸 · 밝기(PWM)로 표현하는 것이 과제의 목표입니다.

> **〔SBC〕 Ubuntu 22.04 GPIO 주의**
> Raspberry Pi OS와 달리 Ubuntu 22.04에서는 `RPi.GPIO`가 정상 동작하지 않을 수 있습니다. 본 과제는 `gpiozero` + **`lgpio`** 백엔드를 표준으로 합니다.
> ```bash
> $ sudo apt install -y python3-lgpio          # lgpio 백엔드 설치
> $ pip install gpiozero lgpio                  # (venv 내에서)
> $ export GPIOZERO_PIN_FACTORY=lgpio           # 핀 팩토리 지정
> ```
> 부팅 설정 경로도 `/boot/config.txt`가 아니라 **`/boot/firmware/config.txt`** 입니다.

## 3. 주제 선택 (3단계 난이도)

아래 중 하나를 선택하거나, 같은 구조를 유지한 변형 주제를 제안서에서 승인받아 진행합니다.

### 🟢 기본
- **이미지 분류 → LED ON/OFF**: 분류 결과가 목표 클래스이면 LED 점등, 아니면 소등.
- 사용 실습: 실습 1~2 (전이학습·이미지 분류), 실습 9 (AI 기반 LED 제어)

### 🟡 심화
- **분류 확신도 → PWM 밝기 제어**: top-1 확신도(confidence)에 비례해 LED 밝기를 0~100%로 연속 제어.
- **객체 탐지 개수 → LED 점멸**: 탐지된 객체 수만큼 LED를 점멸.
- 사용 실습: 실습 3~5 (객체 탐지)

### 🔴 도전
- **세그멘테이션 면적 비율 → 다단계 LED 표시**: 특정 영역이 차지하는 면적 비율을 계산해 LED 밝기 또는 점멸 패턴으로 단계 표시.
- 사용 실습: 실습 6~8 (세그멘테이션)

> 평가는 **선택 난이도 × 완성도**로 이루어집니다. 기본을 완벽히 마치는 것이 심화를 미완성으로 두는 것보다 유리할 수 있습니다.

### 2.2 추론 코드 (`src/infer.py`)
```python
import numpy as np
from PIL import Image
from ai_edge_litert.interpreter import Interpreter

def load_labels(path):
    with open(path) as f:
        return [line.strip() for line in f]

def classify(model_path, label_path, image_path):
    # 모델 로드
    interpreter = Interpreter(model_path=model_path)
    interpreter.allocate_tensors()
    in_det = interpreter.get_input_details()[0]
    out_det = interpreter.get_output_details()[0]

    # 입력 전처리: 224x224, uint8 (quant 모델)
    h, w = in_det['shape'][1], in_det['shape'][2]
    img = Image.open(image_path).convert('RGB').resize((w, h))
    x = np.expand_dims(np.asarray(img, dtype=np.uint8), axis=0)

    # 추론
    interpreter.set_tensor(in_det['index'], x)
    interpreter.invoke()
    out = interpreter.get_tensor(out_det['index'])[0]

    # 결과 해석
    labels = load_labels(label_path)
    idx = int(np.argmax(out))
    conf = float(out[idx]) / 255.0   # quant 출력 정규화
    return labels[idx], conf

if __name__ == "__main__":
    name, conf = classify("models/mobilenet_v2.tflite",
                          "models/labels.txt", "images/cat.jpg")
    print(f"클래스: {name}, 확신도: {conf:.2f}")
```

```
출력 ▶ (예시)

```

### 2.3 LED 제어 코드 (`src/led_control.py`)
```python
import os
os.environ.setdefault("GPIOZERO_PIN_FACTORY", "lgpio")
from gpiozero import PWMLED

led = PWMLED(17)   # BCM 17

def set_brightness(value):
    # value: 0.0 ~ 1.0
    led.value = max(0.0, min(1.0, value))
```

### 2.4 통합 (`src/main.py`)
```python
import sys
from infer import classify
from led_control import set_brightness

TARGET = "cat"   # 목표 클래스 키워드

def run(image_path):
    name, conf = classify("models/mobilenet_v2.tflite",
                          "models/labels.txt", image_path)
    print(f"추론 결과 → {name} ({conf:.2f})")
    if TARGET in name.lower():
        set_brightness(conf)            # 확신도 = 밝기
        print(f"목표 클래스 감지 → LED 밝기 {conf*100:.0f}%")
    else:
        set_brightness(0.0)             # 소등
        print("비목표 클래스 → LED 소등")

if __name__ == "__main__":
    run(sys.argv[1] if len(sys.argv) > 1 else "images/cat.jpg")
```

```
출력 ▶ (예시)
추론 결과 → Egyptian cat (0.78)
목표 클래스 감지 → LED 밝기 78%
# → 실제 LED가 약 78% 밝기로 점등
```

## 5. 트러블슈팅
| 증상 | 원인 | 해결 |
|------|------|------|
| `RuntimeChip` / 핀 접근 오류 | Ubuntu에서 RPi.GPIO 사용 | `GPIOZERO_PIN_FACTORY=lgpio` 설정 |
| LED 밝기 변화 없음 | 일반 GPIO 핀 사용 | `LED` 대신 `PWMLED` 사용 |
| 추론값이 항상 동일 | 입력 dtype 불일치 | quant 모델은 `uint8` 입력 |
| 확신도 > 1.0 | quant 출력 미정규화 | 출력값을 255로 나눔 |

### tar.gz 압축 절차 (Ubuntu 22.04)

```bash
# 1) 제출 폴더 구성 — 이름_학번 디렉터리에 제출물 모으기
$ mkdir -p 홍길동_2025-001
$ cp proposal.md report.md README.md 홍길동_2025-001/   # 제안서·보고서·설명
$ cp -r src/ 홍길동_2025-001/                            # 소스 파일

# 2) tar.gz 압축 (c=생성, z=gzip, v=목록표시, f=파일명)
$ tar -czvf 홍길동_2025-001.tar.gz 홍길동_2025-001/

# 3) 압축 내용 확인
$ tar -tzvf 홍길동_2025-001.tar.gz
```
