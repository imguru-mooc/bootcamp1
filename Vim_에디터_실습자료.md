# Vi / Vim 에디터 실습 — 설치부터 단축키까지

**로봇 플랫폼 (Robot Platform) · 임베디드 리눅스 실습 자료**
운영체제: **Ubuntu 22.04 LTS** (데스크톱 PC · 라즈베리파이 4 · Jetson Orin Nano 공통)

> 임베디드 리눅스에서는 모니터·마우스 없이 **터미널(SSH)만으로** 보드에 접속해 파일을 편집하는 일이 많습니다.
> `vi`는 거의 모든 리눅스에 기본 탑재되어 있어, **어떤 환경에서도 쓸 수 있는 필수 도구**입니다.
> 처음엔 낯설지만 손에 익으면 가장 빠른 편집기가 됩니다. 직접 입력하며 따라 하세요.

---

## 학습 목표

- `vi`와 `vim`의 차이를 이해하고 설치/확인한다.
- Vim의 **모드(mode)** 개념을 이해한다. ← *가장 중요*
- 파일을 열고 → 입력하고 → 저장하고 → 종료하는 전체 흐름을 익힌다.
- 커서 이동, 삭제/복사/붙여넣기, 검색/치환을 단축키로 수행한다.
- 비주얼 모드와 다중 파일 등 유용한 기능을 활용한다.
- `.vimrc`로 개발에 편한 환경을 설정한다.

## 표기 약속

```text
$           # 터미널 프롬프트 (직접 입력하는 명령)
Esc         # 키보드의 Esc 키
:wq         # 마지막 행 모드 명령 — 입력 후 Enter
dd, yy      # 명령 모드에서 누르는 키 (대소문자 구분!)
출력 ▶      # 화면에 나타나는 예시 결과
```

> ⚠️ **대소문자를 구분합니다.** vim에서 `d`와 `D`, `a`와 `A`는 완전히 다른 명령입니다.

## Vi vs Vim

| 구분 | vi | vim (Vi IMproved) |
|---|---|---|
| 의미 | 원조 에디터 | vi의 개선판 |
| 색상 표시(문법 강조) | ✕ | ○ |
| 되돌리기 다중 단계 | 제한적 | ○ |
| 방향키/편집 편의 | 제한적 | ○ |
| 권장 | — | **vim 사용 권장** |

> 💡 Ubuntu에서 `vi`를 입력하면 보통 기본형(vim-tiny)이 실행됩니다.
> 색상·편의 기능을 위해 **vim 정식 버전을 설치**하는 것이 좋습니다.

## 실습 2 · Vim의 모드 이해 ⭐ (핵심)

Vim이 다른 에디터와 가장 다른 점은 **"모드"가 있다**는 것입니다.
지금 어떤 모드인지 모르면 키가 엉뚱하게 동작합니다. 이 개념부터 확실히 잡으세요.

```text
                  i, a, o 등
   ┌────────────┐ ─────────▶ ┌────────────┐
   │  명령 모드  │            │  입력 모드  │
   │  (Normal)  │ ◀───────── │  (Insert)  │
   └────────────┘    Esc     └────────────┘
        │ :
        ▼
   ┌────────────┐
   │ 행 모드     │  (저장/종료/검색치환 등)
   │ (Command)  │
   └────────────┘
```

| 모드 | 역할 | 진입 방법 |
|---|---|---|
| **명령 모드** (Normal) | 커서 이동·삭제·복사 등 *명령 실행* | `Esc` (언제든 복귀) |
| **입력 모드** (Insert) | 실제 *글자 입력* | `i`, `a`, `o` 등 |
| **행 모드** (Command-line) | 저장·종료·검색·치환 | 명령 모드에서 `:` |

> 🔑 **핵심 규칙**
> - vim을 켜면 처음엔 **명령 모드**입니다. 이 상태에서 글자를 입력하면 명령으로 해석되어 화면이 망가집니다.
> - 글을 쓰려면 **반드시 `i`** 등을 눌러 입력 모드로 들어가야 합니다.
> - **헷갈리면 `Esc`** 를 눌러 명령 모드로 돌아오세요. 모든 것의 출발점입니다.

## 실습 3 · 첫 파일 — 열기 → 입력 → 저장 → 종료

```bash
$ vim hello.txt
```

vim이 열리면 다음 순서대로 따라 합니다.

```text
1) i          입력 모드 진입 (화면 아래에 -- INSERT -- 표시됨)
2) 아래 내용을 타이핑:
      Hello, Vim!
      임베디드 리눅스 실습입니다.
3) Esc        명령 모드로 복귀 (-- INSERT -- 사라짐)
4) :wq        저장(write) 후 종료(quit)  →  Enter
```

```bash
# 저장된 내용 확인
$ cat hello.txt
출력 ▶ Hello, Vim!
출력 ▶ 임베디드 리눅스 실습입니다.
```

### 저장 / 종료 명령 정리 (행 모드)

| 명령 | 의미 |
|---|---|
| `:w` | 저장(write) |
| `:q` | 종료(quit) |
| `:wq` 또는 `ZZ` | 저장 후 종료 |
| `:q!` | **저장하지 않고** 강제 종료 |
| `:w 파일명` | 다른 이름으로 저장 |

> 🆘 **빠져나오기가 안 될 때**
> 무조건 `Esc` → `:q!` → `Enter`. (변경 사항 버리고 강제 종료)

> ✅ **체크포인트 3**
> `cat hello.txt` 에 입력한 두 줄이 보이면 통과입니다.

## 실습 4 · 입력 모드 진입 키 & 커서 이동

연습할 파일을 새로 엽니다.

```bash
$ vim practice.txt
```

### 입력 모드 진입 키 (명령 모드에서)

| 키 | 동작 |
|---|---|
| `i` | 커서 **앞**에서 입력 (insert) |
| `a` | 커서 **뒤**에서 입력 (append) |
| `o` | 아래에 **새 줄** 만들고 입력 |
| `O` | 위에 새 줄 만들고 입력 |
| `A` | 줄 **맨 끝**에서 입력 |
| `I` | 줄 **맨 앞**에서 입력 |

### 커서 이동 (명령 모드)

방향키도 되지만, **홈 포지션을 유지하는 hjkl** 사용을 권장합니다.

```text
        k (위)
        ▲
 h ◀────┼────▶ l
(왼쪽)  ▼   (오른쪽)
        j (아래)
```

| 키 | 이동 |
|---|---|
| `h` `j` `k` `l` | 좌·하·상·우 |
| `w` / `b` | 다음 / 이전 **단어**로 |
| `0` / `$` | 줄의 맨 앞 / 맨 끝으로 |
| `gg` / `G` | 문서 맨 위 / 맨 아래로 |
| `숫자G` | 해당 **줄 번호**로 (예: `10G` = 10번 줄) |
| `Ctrl+f` / `Ctrl+b` | 한 화면 아래 / 위로 |

> ✅ **체크포인트 4**
> `o`로 새 줄을 만들어 여러 줄을 입력하고, `gg`·`G`로 문서 맨 위/아래를 오갈 수 있으면 통과입니다.

## 실습 5 · 편집 — 삭제 / 복사 / 붙여넣기 / 되돌리기

모두 **명령 모드(Esc)** 에서 실행합니다.

### 삭제

| 키 | 동작 |
|---|---|
| `x` | 커서 위치 **한 글자** 삭제 |
| `dd` | **현재 줄** 삭제 (잘라내기) |
| `숫자dd` | 여러 줄 삭제 (예: `3dd` = 3줄) |
| `dw` | 커서부터 단어 끝까지 삭제 |
| `D` | 커서부터 줄 끝까지 삭제 |

### 복사(yank) / 붙여넣기(put)

| 키 | 동작 |
|---|---|
| `yy` | 현재 줄 복사 |
| `숫자yy` | 여러 줄 복사 (예: `2yy`) |
| `yw` | 단어 복사 |
| `p` | 커서 **뒤/아래**에 붙여넣기 |
| `P` | 커서 **앞/위**에 붙여넣기 |

> 💡 vim에서 삭제(`dd`)는 사실 **잘라내기**입니다. `dd` 후 `p`를 누르면 다른 위치에 붙여 넣을 수 있어 **줄 이동**에 유용합니다.

### 되돌리기 / 다시하기

| 키 | 동작 |
|---|---|
| `u` | 실행 취소 (undo) |
| `Ctrl+r` | 다시 실행 (redo) |
| `.` | **직전 명령 반복** (매우 유용) |

> 🧪 **실습**: `practice.txt`에서
> ① `yy`로 한 줄 복사 → `p`로 붙여넣기(줄 복제)
> ② `dd`로 줄 삭제 → `u`로 되돌리기
> ③ `3dd`로 3줄 삭제 → `p`로 다른 곳에 붙여넣기

> ✅ **체크포인트 5**
> `yy`+`p`로 줄 복제, `dd`+`u`로 삭제·복원이 되면 통과입니다.

## 실습 6 · 검색 & 치환

### 검색 (명령 모드)

| 키 | 동작 |
|---|---|
| `/단어` | 아래 방향으로 검색 → Enter |
| `?단어` | 위 방향으로 검색 |
| `n` | 다음 결과로 이동 |
| `N` | 이전 결과로 이동 |

```text
예) /Robot   입력 후 Enter  →  'Robot' 찾기
    n        다음 'Robot'으로 이동
```

### 치환 (행 모드 `:`)

```text
:s/찾을것/바꿀것/        현재 줄에서 첫 번째만 치환
:s/찾을것/바꿀것/g       현재 줄 전체 치환
:%s/찾을것/바꿀것/g      문서 전체 치환
:%s/찾을것/바꿀것/gc     문서 전체 치환 (하나씩 y/n 확인)
```

```text
예) :%s/Vim/VIM/g   →  문서 안 모든 'Vim'을 'VIM'으로 변경
```

> 🧪 **실습**: `:%s/리눅스/Linux/g` 로 문서 안 '리눅스'를 모두 'Linux'로 바꿔 보세요.

> ✅ **체크포인트 6**
> `/`로 단어를 검색하고, `:%s///g`로 전체 치환에 성공하면 통과입니다.

## 실습 7 · 비주얼 모드 & 유용한 기능

### 비주얼 모드 — 범위 선택 후 작업

| 키 | 동작 |
|---|---|
| `v` | 글자 단위 선택 |
| `V` | **줄 단위** 선택 |
| `Ctrl+v` | **블록(사각형)** 선택 |

선택 후 `d`(삭제), `y`(복사), `>`(들여쓰기) 등을 적용합니다.

```text
예) V 로 줄 선택 → 방향키(j)로 범위 확장 → d 로 여러 줄 한 번에 삭제
    V 로 줄 선택 → > 로 들여쓰기 / < 로 내어쓰기
```

### 자주 쓰는 추가 명령

| 명령 | 동작 |
|---|---|
| `:set number` / `:set nu` | 행 번호 표시 |
| `:set nonu` | 행 번호 숨기기 |
| `:set paste` | 붙여넣기 시 자동 들여쓰기 방지 |
| `r문자` | 커서의 한 글자를 다른 글자로 교체 |
| `cw` | 단어를 지우고 바로 입력 모드 |
| `>>` / `<<` | 줄 들여쓰기 / 내어쓰기 |
| `Ctrl+v` 블록 선택 | 여러 줄 동시 편집 |

> 🧪 **실습**: `:set nu` 로 행 번호를 켜고, `V`로 3줄을 선택해 `>` 로 한 번에 들여써 보세요.

> ✅ **체크포인트 7**
> `V`(줄 선택) + `d`(삭제)와 `:set nu`(행 번호)가 동작하면 통과입니다.

## 실습 9 · `.vimrc` 설정 (개발 환경 만들기)

vim을 켤 때마다 적용되는 개인 설정 파일을 만듭니다.

```bash
$ vim ~/.vimrc
```

`i`로 입력 모드 진입 후 아래 내용을 작성합니다.

```vim
" ===== 기본 설정 =====
set number          " 행 번호 표시
set autoindent      " 자동 들여쓰기
set smartindent     " 코드 스마트 들여쓰기
set tabstop=4       " Tab 폭 4칸
set shiftwidth=4    " 들여쓰기 폭 4칸
set expandtab       " Tab을 스페이스로 변환
syntax on           " 문법 색상 강조
set hlsearch        " 검색어 강조
set cursorline      " 현재 줄 표시
set mouse=a         " 마우스 사용 허용
```

작성 후 `Esc` → `:wq` 로 저장합니다. 새로 `vim`을 열면 설정이 적용됩니다.

> ⚠️ **〔주의〕** `.vimrc`에서 `set expandtab`(Tab→스페이스)을 켜면,
> **Makefile 편집 시 문제**가 생깁니다. Makefile은 반드시 진짜 Tab이 필요하므로,
> Makefile을 편집할 땐 `:set noexpandtab` 을 적용하세요.

> ✅ **체크포인트 9**
> `.vimrc` 저장 후 새로 vim을 열었을 때 **행 번호와 문법 색상**이 보이면 통과입니다.

## 단축키 치트시트 (요약)

복습용 — 인쇄해서 옆에 두고 쓰세요.

```text
[ 모드 전환 ]
  i a o I A O   입력 모드 진입       Esc   명령 모드 복귀

[ 저장 / 종료 ]
  :w   저장        :q   종료        :wq  저장후종료    :q!  강제종료

[ 커서 이동 ]
  h j k l  좌하상우     w / b  단어     0 / $  줄 앞/끝
  gg / G   처음/끝      숫자G  해당 줄

[ 편집 ]
  x   글자삭제     dd  줄삭제      dw  단어삭제     D  줄끝까지삭제
  yy  줄복사       p / P 붙여넣기   u   되돌리기     Ctrl+r 다시하기   . 반복

[ 검색 / 치환 ]
  /단어  검색      n / N  다음/이전
  :%s/찾기/바꾸기/g   전체 치환     :%s/.../gc  확인하며 치환

[ 비주얼 / 기타 ]
  v / V / Ctrl+v   선택(글자/줄/블록)
  :set nu  행번호    >> / <<  들여쓰기/내어쓰기
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 글자를 쳤더니 화면이 망가짐 | 명령 모드에서 입력함 | `Esc` 후 `u`로 되돌리기 → `i`로 입력 모드 진입 |
| 종료가 안 됨 | 저장 안 된 변경 사항 | `Esc` → `:q!`(버리고 종료) 또는 `:wq`(저장후 종료) |
| 색상이 안 나옴 | vim-tiny 사용 중 | `sudo apt install -y vim` 후 사용 |
| 붙여넣기 시 계단식 들여쓰기 | 자동 들여쓰기 동작 | `:set paste` 후 붙여넣기 → 끝나면 `:set nopaste` |
| 방향키가 `ABCD`로 입력됨 | 호환 모드 vi-tiny | vim 정식 설치 또는 hjkl 사용 |
| Makefile이 자꾸 깨짐 | expandtab으로 Tab이 스페이스됨 | `:set noexpandtab` 적용 후 편집 |

- [ ] 파일을 `:wq`로 저장·종료, `:q!`로 강제 종료할 수 있다
- [ ] `dd`(삭제)·`yy`+`p`(복사·붙여넣기)·`u`(되돌리기)를 사용할 수 있다

- [ ] `V`(비주얼)로 여러 줄을 선택해 작업할 수 있다
